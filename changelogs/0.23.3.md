# v0.23.3 - 2026-05-30

## 前情与承接

`v0.23.3` 承接 `v0.23.2` 的 SSE 智能分段推送，完成 Phase 3 的交互闭环：群聊 @ 触发验证和危险操作确认交互（用户回复"确认"/"取消" → CaiBao confirm API → 结果回推）。

## 版本概览

1. **危险操作确认交互闭环** — SSE streaming / 同步降级双路径均支持 `confirmation.required` → 用户回复确认 → 调用 `POST /api/v1/agent/runs/{run_id}/confirm` → 结果推送回 QQ。
2. **关键词匹配安全** — `_match_keyword()` 函数：英文短词（ok/no/yes）精确匹配防止误触发，中文词（确认/取消/是/否）子串匹配支持自然语言。
3. **群聊确认隔离** — 待确认状态按 `chat_id` 隔离，私聊和群聊各自独立。

## 主要更新

### 确认交互闭环

```
SSE: confirmation.required → _save_pending_confirmation → QQ 提示
  → 用户 "确认" → _try_handle_confirmation → confirm_run → 结果回推
  → 用户 "取消" → _handle_cancel → 清除状态
  → 用户发新消息 → pending auto-dismiss → 正常处理新消息
```

- `_try_handle_confirmation`：入站消息拦截器，检查是否有待确认操作
- `_handle_confirm`：调用 `confirm_run` API + finally 保证清理
- `_handle_cancel`：清除待确认 + 通知用户
- `PendingConfirmation`：run_id / tool_name / arguments / created_at
- 同步降级路径 `_handle_message_sync` 同样支持

### 关键词匹配 `_match_keyword()`

- `_EXACT_MATCH_KEYWORDS = {"ok", "no", "yes"}` — 英文短词需精确匹配
- 中文词（确认/取消/是/否/执行/跳过/不执行）— 子串匹配
- 大小写不敏感 + 前后空格 trim

### 群聊支持

- 群聊已在 NapCatChannel 中实现（Phase 2），本次新增：
  - 群聊 conversation_id 格式 `qq_group_{group_id}`
  - 群聊确认隔离：不同群/私聊的待确认互不干扰
