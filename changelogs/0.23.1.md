# v0.23.1 - 2026-05-30

## 前情与承接

`v0.23.0` 承接 `v0.22.0` 的插件系统集成工作，主要完成两项内容：

1. **修复 Phase 4 遗漏** — `v0.22.0` 提交中 `deps.py` 已向 `AgentService` 和 `ToolService` 构造函数传递 `plugin_manager` 参数，但两个服务的构造函数本身未添加该参数，导致 Agent App 调用链路 400 报错。本次补齐两个构造函数，完成插件系统的闭合集成。
2. **QQ Bot 适配器** — 参考 `akashic-agent` 的 MessageBus + Channel 架构，新增独立的 `qqbot_adapter/` 模块，通过 NapCat / OneBot v11 WebSocket 协议接入 QQ，与 CaiBao Agent API 双向桥接。

## 版本概览

1. **QQ Bot 适配器（Phase 1）** — MessageBus 消息总线 + NapCatChannel（OneBot v11 WebSocket）+ AgentBridge（CaiBao API SSE 桥接），私聊白名单、群聊 @ 触发、流式分段推送、危险操作确认交互。
2. **Phase 4 插件系统集成补齐** — `AgentService.__init__` 和 `ToolService.__init__` 增加 `plugin_manager` 参数，完成 BeforeTurn / PromptRender / BeforeReasoning / AfterReasoning / AfterTurn 全链路 hook 闭合。
3. **channel 体系扩展** — `chat_history_service` 新增 `"qqbot"` channel；`agent_service._record_agent_history` 改为使用实际 `trigger_channel` 而非硬编码 `"agent"`。

## 主要更新

### QQ Bot 适配器

```
QQ消息 → NapCat WS → NapCatChannel → MessageBus.inbound
    → AgentBridge → CaiBao API (SSE)
    → MessageBus.outbound → NapCatChannel → QQ回复
```

- **MessageBus**（`core/bus.py`）：异步入站/出站队列 + 订阅分发 + 首次失败 2s 后重试 + 重试仍失败消息丢弃（不阻塞后续）。
- **NapCatChannel**（`channels/napcat_channel.py`）：OneBot v11 WebSocket 客户端；私聊白名单过滤；群聊 `require_at` + 成员白名单；CQ 码纯文本提取；WebSocket 指数退避自动重连（base 1s, max 60s, backoff 2x）；API 调用 echo 匹配 Future 响应。
- **AgentBridge**（`core/bridge.py`）：JWT cookie 登录认证；`POST /api/v1/agent/runs` 创建运行；SSE 流式消费；`llm.delta` 累积分段推送（每 300 字符）；tool.* 事件实时转发；`confirmation.required` 危险操作确认提示；`_split_text` 按换行边界分段（单条 3500 字符限制）。
- **会话隔离**：私聊 `qq_{user_id}`，群聊 `qq_group_{group_id}`，每个 QQ 用户/群对应独立 CaiBao conversation。
- **41 个单元测试**：覆盖 SSEEvent 解析（10）、MessageBus 队列/订阅/重试/幂等（9）、文本分段/会话映射（10）、Markdown→QQ 转换/确认提示（7）、会话 ID 生成（5）。

### Phase 4 插件集成补齐

- `AgentService.__init__` 新增 `plugin_manager` 参数；同步和流式全路径已接入 6 个 Phase hook（`v0.22.0` 完成），本次补齐构造函数接收参数并存储 `self.plugin_manager`。
- `ToolService.__init__` 新增 `plugin_manager` 参数；`execute()` 中 `check_tool_pre()` 前置钩子已接入（`v0.22.0` 完成），本次补齐构造函数。
- `_record_agent_history` 使用 `payload.trigger_channel` 而非硬编码 `"agent"`，确保 QQ Bot 等外部通道的消息正确标记。

## 兼容性说明

- `AgentService` 和 `ToolService` 构造函数的 `plugin_manager` 参数为可选（默认 `None`），不影响已有调用方。
- `chat_history_service` channel 白名单新增 `"qqbot"`，不影响已有 channel（echo/ask/action/agent）。
- QQ Bot 适配器为独立目录 `qqbot_adapter/`，不修改 CaiBao 主应用代码，可独立部署。
