## QQ 官方 Bot 通道

```
认证: POST /app/getAppAccessToken (app_id + client_secret)
  → access_token (7200s, 提前 300s 自动刷新)

收消息: wss://api.sgroup.qq.com/websocket
  op=10 Hello → heartbeat_interval
  op=0  Dispatch → t 在顶层（非 d 内部），分发到 handler
  op=1/11 Heartbeat → 保活

发消息: HTTP POST
  私聊: /v2/users/{openid}/messages
  群聊: /v2/groups/{group_openid}/messages
  图片: POST /v2/{target}/{id}/files (multipart form)
```

- 私聊：C2C_MESSAGE_CREATE 事件 → 图片附件自动下载
- 群聊：GROUP_AT_MESSAGE_CREATE 事件 → `<@!xxx>` 清理
- 群聊 `group_openid` → `g{openid}` chat_id 格式（与 NapCat 统一）

## 图片工具 (`image_utils.py`)

- `extract_image_urls_from_cq()`：`[CQ:image,url=...]` + HTTP URL 双格式提取
- `download_images_from_urls()`：异步下载 + 10MB 上限 + 单张失败静默跳过
- `download_images_from_message()`：消息文本 → 提取 URL → 下载
- `guess_mime_type()`：扩展名 → MIME
- `build_data_uri()`：字节 → `data:image/...;base64,...`

## 安全修复

### P0: 新 QQ 用户首条消息 404
- 根因：`qq_{user_id}` 作为 deterministic conversation_id 传给 `/api/v1/agent/runs`，后端 `document_service.resolve_space_id()` 强制查 Conversation 是否存在 → 新人 404
- 修复：`_ensure_conversation()` 方法，运行前先 `POST /api/v1/conversations` 创建真实会话，映射缓存在 `_conversation_cache`

### P1: 默认配置安全风险
- 根因：NapCat 默认 `enabled=true` + `allow_from=[]`，空白名单 = 放行所有人
- 修复：默认 `enabled=false`；新增 `allow_all` 开关，`allow_all=false` 且白名单为空时拒绝所有人

### P1: 群聊危险操作确认可被其他成员接管
- 根因：`PendingConfirmation` 只按 `chat_id` 存，不记录发起人
- 修复：新增 `requester_user_id` 字段；`_try_handle_confirmation` 校验发起人身份，非发起人消息走正常流程不接管

### P1: JWT Token 15 分钟过期不刷新
- 根因：`AgentBridge` 登录后 `_authenticated=True` 永不刷新
- 修复：`_api_post_with_retry()` 包装，401 时自动调 `/auth/refresh`，失败则重新登录

### P2: 流式路径不写 ChatHistory
- 根因：`AgentService.stream_run()` 到 `run.completed` 就返回，不调用 `_record_agent_history()`
- 修复：`stream_run()` 完成后调用 `_record_agent_history()`
