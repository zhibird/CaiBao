# v0.24.2 - 2026-06-02

## 前情与承接

`v0.24.2` 承接 `v0.24.1` 的双人格 Persona 系统，本版本聚焦联网搜索模块的**工程质量提升**——参照 akashic-agent 实现，将 web_fetch 和 web_search 从 MVP 级重写为生产级。

## 版本概览

1. **web_fetch 重写** — 从正则 HTML 解析升级为 lxml DOM + html2text，补齐 User-Agent、Accept 协商、二进制检测
2. **web_search 增强** — 暴露 Exa 全部参数（num_results / livecrawl / type），结果解析器用空行分段状态机替代逐行猜测
3. **SSRF 防护加固** — 新增 `_validate_url_target` 快速预检层，修复 localhost 漏拦 bug

## 主要变更

### web_fetch — 对齐 akashic-agent 实现

| 维度 | 旧实现 | 新实现 |
|------|--------|--------|
| HTML→文本 | 正则 regex | lxml DOM 解析，精确移除 script/style/noscript/iframe/object/embed |
| HTML→Markdown | 不支持 | html2text 转换 |
| User-Agent | 无 | `caibao/1.0` |
| Accept 协商 | 无 | markdown/text/html 三档 Accept header |
| 二进制检测 | 无 | pdf/image/video/audio 前缀匹配拒绝 |
| 大小检查 | 仅流式读取截断 | Content-Length 预检 + 实际 body 双重 5MB 限制 |
| 输出格式 | `{title, status_code, final_url, content, truncated}` | `{url, final_url, status, content_type, format, length, text, truncated, note}` |
| 参数 | `max_chars` | `format`（text/markdown/html）+ `timeout` |
| 字符上限 | 参数控制 | 固定 50K chars（~12K tokens） |

### web_search — 参数对齐 + 解析器重写

**新暴露的 Exa 参数：**
```json
{
  "num_results": 8,           // 默认 8，最大 20
  "livecrawl": "fallback",    // fallback（缓存优先）/ preferred（实时优先）
  "type": "auto"              // auto（均衡）/ fast（快速）/ deep（深度）
}
```

**_parse_exa_results 重写：**
- 旧：逐行状态机，假设 Title 一定最先出现，非预期行静默丢失
- 新：空行分段 + Title 开新 block + 未知字段追加到 snippet

### SSRF 防护加固

- 新增 `_validate_url_target()` 纯 IP 快速预检（无 DNS 依赖），与 `_host_is_dangerous()` DNS 深度检查形成双层防护
- 修复 localhost / 0.0.0.0 在预检层漏拦的 bug
