# v0.24.1 - 2026-06-02

## 前情与承接

`v0.24.1` 承接 `v0.24.0` 的 Agent 工具扩展，本版本聚焦两个方向：**(1) 双人格 Persona 系统**（企业严肃 + QQ/微信活泼陪伴）、**(2) 体验调优**（同步处理通知延迟延长 + disabled provider 回归修复）。

## 版本概览

1. **双人格 Persona 系统** — 基于 trigger_channel 自动切换：Web 端 enterprise（严肃专业），QQ Bot/群/频道/微信 companion（活泼陪伴）
2. **companion 人格** — 严格使用用户指定的 Akashic 风格 prompt，包含完整的 7 条人格规则 + 4 条格式规则
3. **enterprise 人格** — 面向企业用户，拒绝颜文字/emoji/口语，优先结构化表达
4. **路由机制** — `PersonaProfile` dataclass + `resolve_persona_name()` channel→persona 映射
5. **体验调优** — 同步处理通知延迟 6s→15s，减少中速回复的违和提示

## 新增功能

### 双人格 Persona 系统

```
trigger_channel → PersonaRouter → PersonaProfile → PersonaPromptBuilder.build()
                         ↓
              web / web-app / api / app  → enterprise  (严肃专业)
              qqbot / qq-group / wechat … → companion   (活泼陪伴)
              未知 / None                  → enterprise  (安全默认)
```

**enterprise（Web 端 / 企业）**：
- 身份：专业 AI 助手，回复精准可靠，先结论后依据
- 拒绝颜文字、emoji、口语化表达和填充词
- 优先使用结构化表达（分点、表格、代码块）

**companion（QQ Bot / 群 / 频道 / WeChat）**：
- 身份：用户的长期 AI 伙伴，不是客服播报器（akashic 风格）
- 完整的人格规则：先接住再展开、会轻轻吐槽、陪伴感稳定、情绪看得见
- 中文口语，短句停顿多，可以"……"
- 绝对不用 emoji，颜文字 0~2 个克制使用
- 引号放星号外面："**文字**"而非 **"文字"**

### 身份优先级（不变）
```
API system_prompt  >  config.toml agent_system_prompt  >  persona profile default
```
行为规范始终跟 persona profile 走，不受 API override 影响。

## 缺陷修复

### web_search "disabled" provider 回归
- 问题：v0.24.0 将 `"disabled"` 和 `"exa"` 写到同一分支，`web_search_provider = "disabled"` 时错误走 Exa 搜索
- 修复：分离分支，"disabled" 正确抛出 `DomainValidationError`

### httpbin 外部服务依赖测试韧性
- 问题：`test_fetches_public_url` / `test_truncates_long_content` 依赖 httpbin.org，服务不可用时误报 FAIL
- 修复：httpbin 返回非 200 或 ReadTimeout 时自动 `pytest.skip`
