# v0.20.0 - 2026-05-27

## 前情与承接

`v0.20.0` 承接 `v0.19.0` 的 Agent 引擎现代化 Phase 1（LLM Function Calling + 多轮工具循环 + SSE 流式 + 模型路由），完成 Phase 2：工具生态与 MCP。

Phase 2 把 CaiBao 的工具系统从“少量内置业务工具”升级为“内置通用工具 + MCP 动态工具 + 多层安全治理”的工具生态。

## 版本概览

1. **ToolCatalogService** — 统一工具目录，合并 builtin + generic（web/file/shell）+ MCP 三类来源，统一提供 `list_definitions` / `get_definition` / `get_openai_tools`。
2. **ToolSafetyService** — 多层安全治理：schema 校验、enabled_tools 白名单、dangerous 确认、循环守卫（单 run 最多 20 次/单工具最多 8 次/连续重复最多 3 次）、输出截断、审计日志。
3. **MCP stdio client** — JSON-RPC 2.0 over stdio，实现 `initialize → notifications/initialized → tools/list → tools/call` 完整生命周期，含并发锁和超时看门狗。
4. **通用 Web 工具** — `web_fetch`（DNS 解析 + ipaddress 全类型 SSRF 防护、逐跳手动 redirect、流式 max_bytes 截断）、`web_search`（Brave / Tavily 抽象）。
5. **通用 File 工具** — `list_dir` / `read_file` / `write_file` / `edit_file`，含 path sandbox（resolve → relative_to 防逃逸）、symlink 阻断。
6. **通用 Shell 工具** — `shell_exec` / `shell_status` / `shell_kill`，含破坏性命令黑名单、网络命令阻断、元字符拦截、Windows cmd 内建命令回退、后台任务内存注册表。
7. **Agent 执行闭环接入** — `_build_tools()` 使用 catalog，`_tool_loop()` / `_tool_loop_stream()` 通过 `ToolService.get_tool_definition()` 查找 tools，`ToolService.execute()` 统一分发 builtin / generic / MCP。
8. **危险工具确认收口** — Agent 双检 + `ToolService.execute()` 拒绝未确认 + dry_run 短路不执行 handler + `/chat/action` 拒绝 generic/MCP 危险工具 + MCP test 强制 dry_run（非 admin）。
9. **MCP API** — `GET /mcp/servers`、`POST /mcp/servers/reload`、`GET /mcp/tools`、`POST /mcp/tools/{tool_name}/test`。

## 主要更新

1. 新增 `ToolCatalogService`：聚合 builtin + generic + MCP 工具定义，支持 `enabled_tools` 过滤和 OpenAI tools schema 输出。
2. 新增 `ToolSafetyService`：preflight（工具存在 → 白名单 → schema → 循环守卫 → 危险确认）→ postflight（输出截断）→ record_call（审计）。
3. 新增 `MCPStdioClient`：启动 subprocess、JSON-RPC 握手、工具发现、工具调用、超时看门狗、并发锁保护。
4. 新增 `MCPManager`：配置加载（含 `${ENV_VAR}` 展开）、server 生命周期管理、命名空间映射（`mcp__{server}__{tool}`）、工具发现与过滤。
5. 新增 `web_tools.py`：`web_fetch`（SSRF DNS 校验 + 逐跳 redirect + 流式截断）、`web_search`（Brave / Tavily）。
6. 新增 `file_tools.py`：`list_dir` / `read_file` / `write_file` / `edit_file`，path sandbox + symlink 阻断。
7. 新增 `shell_tools.py`：`shell_exec` / `shell_status` / `shell_kill`，安全黑名单 + 元字符阻断 + Windows 回退。
8. `AgentService` 改造：`_build_tools()` 接入 catalog，`_tool_loop()` / `_tool_loop_stream()` 使用 `ToolService.get_tool_definition()`。
9. `ToolService` 改造：统一分发 builtin / generic / MCP 三种来源，dry_run 短路返回，所有执行走 safety preflight。
10. `ActionChatService` 加固：拒绝 generic / MCP 危险工具，仅允许 Phase 1 内置工具走快速通道。
11. 新增 MCP 路由和 test 端点，test 对危险工具强制 dry_run（非 dev-admin）。
12. `MCPServerConfig` 名称校验：禁止 `__`、`.`、空格，限制 `[a-zA-Z0-9_-]+` 且 namespaced 组合名 ≤ 64 字符。

## 安全设计

- **dry_run**：`ToolService.execute(dry_run=True)` 在所有真实派发前短路返回，handler 不会被调用。
- **dangerous 确认**：危险工具默认 `requires_confirmation=True`，Agent 暂停等用户确认；非 Agent 入口（chat/action、MCP test）均有限制。
- **SSRF 防护**：`socket.getaddrinfo()` DNS 解析 → `ipaddress` 检查 private / loopback / link-local / reserved / multicast / unspecified + metadata IP。
- **Path sandbox**：`resolve()` → `relative_to(root)`，拒绝 `../` 逃逸和 symlink。
- **Shell 安全**：破坏性正则黑名单 + 网络命令阻断 + 元字符 `;` `&` `|` `` ` `` `$(` `${` `>` `>>` 拦截 + `shell=False`（Windows 内建命令回退 `cmd /d /c`）。
- **MCP 命名空间**：server name 与 tool name 均校验 `[a-zA-Z0-9_-]+`，组合名 ≤ 64 字符。
- **MCP 并发**：per-client `threading.Lock` 串行化 send/receive，防止响应交叉吞并。

## 处理链路

```
AgentRunRequest
  → AgentService.run() / stream_run()
    → _build_tools() → ToolCatalogService.get_openai_tools()
    → _tool_loop() / _tool_loop_stream()
      → LLM 返回 tool_calls
      → ToolService.get_tool_definition() → catalog 查找（builtin → generic → MCP）
      → ToolService.execute()
        → ToolSafetyService.preflight()
          → tool 存在 → enabled → schema 校验 → loop guard → dangerous 确认
        → dry_run? → 短路返回
        → dispatch: builtin handler / generic handler / MCP handler
        → ToolSafetyService.postflight_output()
        → ToolSafetyService.record_call()
      → 工具结果追加到 messages → loop
```
