# v0.27.0 - 2026-07-11

## 前情与承接

本版本承接 `v0.26.0`（CLI 编码代理后端：Claude Code / Codex / Pi 三后端接入 Agent 引擎）的发布收尾：

1. `v0.26.0` 完成了三后端的完整链路（子进程骨架、事件映射、多轮续接、SSE 流式、轨迹持久化）并附带 13 项对抗审查修复；
2. 当时遗留的边界：`run_mode` 只在 App 层校验，直连 `/agent/runs` 的未知值会静默降级；模式白名单在四处手抄；Docker Compose 未透传 `CLI_AGENT_*` 环境变量，容器内即使扩展镜像也无法启用该功能；
3. 本次以发布前全量一致性审计（5 维度：changelog 对代码 / 文档对代码 / 配置-环境-Compose / 仓库卫生 / 前后端契约）驱动，把上述缺口全部闭环，并完成 0.27.0 版本对齐。

## 版本概览

本版本主要完成：

1. **未知 `run_mode` 显式拒绝**——直连 Run API 传入拼写错误的模式（如 `claude-code`）现在返回 400 并列出合法值，不再静默落入内置 `agent_auto` 循环用服务端 LLM 执行；
2. **模式白名单单一事实来源**——`AgentAppService._ALLOWED_MODES` 与 Run 层校验均从 `CLI_AGENT_MODES` 推导，新增 CLI 后端时不再需要多处手改；
3. **Docker Compose 透传 `CLI_AGENT_*`**——补齐 16 个环境变量条目（默认关闭），容器内工作区默认 `/data/cli_agent_workspaces`；修正 `APP_VERSION` 兜底值（0.19.0 → 0.27.0）；
4. **仓库卫生**——解除 10 个误跟踪的运行期日志文件（`output/*.log`、`.superpowers/**/state/*.log`），`.gitignore` 新增 `*.log` 与 `output/`；清理两处死导入。

## 问题现象 → 根因 → 修复

1. **`run_mode` 静默降级**
   - 现象：`POST /agent/runs` 传 `"run_mode": "claude-code"`（连字符笔误）时，run 正常执行并返回内置 FC 循环的结果，用户以为在用 CLI 代理；
   - 根因：`AgentRunRequest.run_mode` 是自由字符串，`run()` / `stream_run()` 的分发是 `if in CLI_AGENT_MODES … elif workflow … else agent_auto`，未知值走 else 分支；App 层的 `_ALLOWED_MODES` 校验管不到直连 Run API 的请求；
   - 修复：`AgentService.start()` 与 `run()` 入口新增 `_validate_run_mode()`，白名单外抛 `DomainValidationError` → 路由层 400，错误消息列出全部合法值（`app/services/agent_service.py`）。
2. **Compose 环境变量断链**
   - 现象：按文档扩展镜像打包 CLI 后，宿主 `.env` 设 `CLI_AGENT_ENABLED=true` 容器内仍不生效；
   - 根因：`docker-compose.yml` 采用显式 `environment:` 白名单且无 `env_file:`，未列出的变量不进容器，而 0.26.0 未把 `CLI_AGENT_*` 加入清单；
   - 修复：补齐全部 16 个 `CLI_AGENT_*` 透传条目并注释说明镜像前提；`docs/step-13-cli-agent-backends.md` 第 7 节同步补充说明。
