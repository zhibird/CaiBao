# v0.21.0 - 2026-05-28

## 前情与承接

`v0.21.0` 承接 `v0.20.0` 的工具生态与 MCP 阶段，完成 Phase 3：多层记忆架构、事件总线、检索增强。把 CaiBao 的记忆系统从"纯向量检索"升级为"Markdown 可读记忆 + 向量可检索记忆 + 事件驱动后处理 + 查询增强"的完整记忆架构。

## 版本概览

1. **EventBus** — 类型化事件总线，支持 `on`（拦截链）/ `observe`（通知）/ `fanout`（并发）/ `enqueue`（后台队列），线程安全，emit 自动通知 observers，双检锁 singleton。
2. **Markdown 记忆文件层** — `data/memory/{hash}/{hash}/{hash}/` 下 MEMORY.md（长期事实/偏好）、HISTORY.md（时间线附加）、RECENT_CONTEXT.md（近期上下文）、PENDING.md（待归档），全部原子写入，幂等 source_ref marker。
3. **Agent Turn 生命周期** — TurnStarted / RetrievalCompleted / ToolCalled / ToolResulted / StepCompleted / TurnCommitted / MemoryUpdated / ConsolidationCommitted 八个事件覆盖 sync/stream/confirm/workflow/fallback 全路径。
4. **Consolidation 记忆沉淀** — 每 N 轮对话触发一次 LLM 提取（多轮窗口上下文），产出 history_entries（追加 HISTORY.md）、pending_items（追加 PENDING.md，identity/preference 标签同步到 MEMORY.md）、compression/ongoing_threads（更新 RECENT_CONTEXT.md）。
5. **向量桥接** — `MemoryService.create_from_consolidation()` 接收 ConsolidationCommitted 事件，逐条 entry 写入 MemoryCard + 自动 embedding，per-entry source_ref 幂等去重，独立 DB session 避免请求生命周期泄漏。
6. **Prompt 注入** — Agent 运行时自动将 MEMORY.md（≤6000 chars）和 RECENT_CONTEXT.md（去掉 Recent Turns，≤3000 chars）注入 system prompt，PENDING.md / HISTORY.md 不注入。尊重 `include_memory=false` 开关。
7. **检索增强** — QueryRewriter（fast 路由改写查询）、HyDEEnhancer（生成假设文档）、SufficiencyChecker（二次检索），全 fail-open 降级，merge+sort+truncate 保证 top_k 契约，fast_timeout=5s 独立控制延迟。
8. **DB Migration** — `memory_cards` 新增 `source_ref`（VARCHAR 128）+ `memory_type`（VARCHAR 32, default "card"），复合唯一索引 `(team_id, user_id, space_id, source_ref, memory_type)`。

## 安全设计

- **路径逃逸防护**：用户/团队/空间 ID 经 SHA-256 hash 映射为目录名，`..` / `/` / `\` 等字符在 hash 后消除。
- **原子写入**：所有文件写入走 `tempfile + os.replace`，崩溃不破坏原文件。
- **幂等 source_ref**：HISTORY.md / PENDING.md 追加使用 `<!-- caibao:source_ref:{hash}:kind:{kind} -->` 隐藏标记，同一 source_ref 不重复写。
- **后台 DB session 隔离**：向量桥接通过 `SessionLocal` 创建独立 session，finally 关闭，不持有请求级 session。
- **内存注入控制**：`include_memory=false` 时完全跳过 Markdown 记忆注入，与旧 vector memory card 行为一致。
- **fail-open**：增强检索和 LLM 提取任何环节失败都静默降级，不阻塞 Agent 主回答。

## 事件流

```
AgentService.run() / stream_run() / confirm_run()
  → emit(TurnStarted)
  → _record_retrieval_step() → emit(RetrievalCompleted)
  → _tool_loop() / _tool_loop_stream() / _execute_tool_calls()
      → emit(ToolCalled) → execute → emit(ToolResulted)
  → _record_step() → emit(StepCompleted)
  → _finish_run(status=completed|partial_success)
      → enqueue(TurnCommitted)
        → [EventBus worker]
          → MemoryConsolidationService._on_turn_committed()
            → _update_recent_turns() → RECENT_CONTEXT.md
            → 达阈值? → _run_consolidation()
              → LLM 提取 → HISTORY.md / PENDING.md / MEMORY.md
              → observe_event(ConsolidationCommitted)
                → MemoryService.create_from_consolidation() → vector cards
```

## 内存模型

```
Markdown 文件层 (人类可读 / prompt cache 友好)
  MEMORY.md        ← 长期事实/身份/偏好 (注入 prompt)
  RECENT_CONTEXT.md ← 近期上下文摘要 (注入 prompt, Recent Turns 去掉)
  HISTORY.md       ← 事件时间线 (不注入 prompt)
  PENDING.md       ← 待归档缓冲 (不注入 prompt)
         ↓
向量层 (语义检索)
  MemoryCard (category=history, source_ref={ref}:{i})
  MemoryCardEmbedding (自动 embedding)
```
