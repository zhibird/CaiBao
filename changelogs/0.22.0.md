# v0.22.0 - 2026-05-29

## 前情与承接

`v0.22.0` 承接 `v0.21.0` 的多层记忆架构 + 事件总线 + 检索增强，完成 Phase 4：插件系统、主动推送、Drift 空闲任务、SubAgent/PeerAgent/DAG 工作流。把 CaiBao 从"被动 Agent App 平台"升级为"可主动巡检、可插件扩展、可委派子任务、可后台自改进"的企业 Agent 平台。

所有后台能力默认关闭，必须通过配置显式开启。

## 版本概览

1. **Plugin System** — 6 个 Phase hook（BeforeTurn / BeforeReasoning / PromptRender / ToolExecution / AfterReasoning / AfterTurn）接入 AgentService 同步与流式全路径；`@on_tool_pre` 安全拦截 + `@on_event` EventBus 观察者；PhaseModule slot 依赖拓扑排序；PluginManager 扫描/加载/回滚。
2. **Proactive Push** — `ProactiveGateway` 通过 MCP 拉取 alert/content/context 事件；`ProactiveEnergy` 三曲线评分（urgency + relevance − fatigue）自适应 tick 间隔；`OutboundService` database/webhook 双通道发送；`ProactiveScheduler` 接入 lifespan 后台线程。
3. **Drift 空闲任务** — `DriftStateStore` 扫描 `data/drift/` SKILL.md（YAML frontmatter）；`DriftService.run_drift()` LLM mini tool loop；6 个内置 drift 工具（read/write/edit/list work files + message_push + finish_drift）；contextvars 并发状态隔离；3 个内置 seed skill。
4. **SubAgent / PeerAgent / DAG** — `SubAgentService` sync/async 两种模式，profile 限制工具集（research/ops）；`spawn_subagent` 工具注册到 Agent tool catalog；`WorkflowDAGExecutor` 拓扑排序 + 简单条件（equals/contains/exists）。
5. **安全加固** — Drift 路径穿越全入口 `_validate_name` + SHA-256 hash ID；Proactive 租户隔离 team_id 过滤；Plugin tool hook raise=deny 语义；feature flags 全路由 gate；MCP 非破坏性单连接。

## 安全设计

- **Plugin tool hook**：`@on_tool_pre` raise → deny 执行（安全拦截边界），异常不吞。
- **Plugin 隔离**：每个 plugin import 前清空全局 decorator registry，防止跨插件 hook 泄漏。
- **Drift 路径逃逸**：全部入口 `_validate_name()` 正则校验 + SHA-256 hash ID 映射目录名。
- **Drift 并发**：`contextvars.ContextVar` 每调用独立 state，无跨请求污染。
- **Proactive 租户隔离**：`run_tick` team_id 过滤；`OutboundService.retry` team_id 校验。
- **Feature flags**：proactive/drift 全路由 gate，`enabled=false` 返回 403。
- **MCP 非破坏连接**：`ProactiveGateway` 检查 `_servers` 已连接，避免 `connect_all → shutdown_all` 破坏全局连接。

## 处理链路

```
AgentService.run() / stream_run()
  → _run_plugin_phase("BeforeTurn")
  → _build_agent_messages()
  → _run_plugin_phase("PromptRender")
  → _run_plugin_phase("BeforeReasoning")
  → _tool_loop()
    → ToolService.execute()
      → plugin_manager.check_tool_pre()  ← @on_tool_pre hooks
      → dispatch
  → _run_plugin_phase("AfterReasoning")
  → _finish_run() → enqueue(TurnCommitted)
  → _run_plugin_phase("AfterTurn")

ProactiveScheduler (lifespan, 默认 OFF)
  → ProactiveService.run_tick()
    → ProactiveGateway.pull_events()  ← MCP call_tool
    → ProactiveEnergy.score() → decide push
    → OutboundService.send() → database / webhook

Drift (手动触发, POST /drift/run)
  → DriftStateStore.get_skill() → parse SKILL.md
  → DriftService.run_drift()
    → LLM tool loop (drift tools only)
    → finish_drift → persist state.json
```
