# v0.23.2 - 2026-05-30

## 前情与承接

`v0.23.2` 承接 `v0.23.1` 的 QQ Bot 适配器 Phase 1 基础框架，完成 Phase 2（NapCatChannel 私聊实现完善+测试）和 Phase 3（SSE 流式智能分段推送），并修复 Phase 3 审查中发现的 double-send 和频控绕过 bug。

## 版本概览

1. **Phase 2 完善** — NapCatChannel 私聊链路补全单元测试（23 个），覆盖 CQ 码提取、@检测、白名单过滤、群配置解析、echo 格式等核心逻辑。
2. **Phase 3 SSE 智能分段** — 流式推送从固定 300 字符切割升级为句子/段落边界自然分段（三段式：段落 > 句子 > 硬截断），增加频控、去重、输入提示。
3. **Bug 修复** — `_dispatch_sse_event` 的 step_completed/run_completed 导致最终答案 double-send；所有事件 handler 绕过频控直接发 Bus。

## 主要更新

### NapCatChannel 测试完善 (Phase 2)

- CQ 码提取测试（9）：纯文本/图片/AT/表情/未知码/多重码/空消息/畸形/URL 含特殊字符
- @机器人检测测试（5）：CQ 码格式/纯文本 @/@ 其他人/无 @/空 self_id
- 白名单过滤测试（3）：空白名单通过/黑名单拦截/白名单通过
- 群配置解析测试（3）：多群配置/空 ID 跳过/None 默认空 dict
- Chat ID 路由测试（2）：g 前缀→群聊 / 无 g→私聊
- Echo 格式测试（1）

### SSE 流式智能分段 (Phase 3)

- **`find_flush_point()` 三段式算法**：段落结束（\n\n）> 句子结束（。！？!?\n）> 硬截断（空格/700 字符）
- **输入提示**：流式开始即发送「⏳ 正在思考...」
- **频控**：同 chat_id 最小 0.3s 发送间隔，`_send_throttled()` 统一入口
- **去重**：已流式推送的文本不重复发送（子串匹配）
- **格式化**：流式文本应用 `markdown_to_qq()` 转换 Markdown 语法
- 新增 `find_flush_point` 测试 10 个 + 频控测试 3 个 + 去重测试 4 个

### Bug 修复

1. **Double-send**：`_dispatch_sse_event` 中 step_completed/run_completed 的 handler 发送最终答案，同时 `_handle_message_streaming` 的事后去重逻辑也发送 → 改为 early return，统一由 streaming 方法发送。
2. **频控绕过**：`_dispatch_sse_event` 全部事件 handler 直接调 `self.bus.publish_outbound()`，不经过频控 → 全部改为 `await self._send_throttled()`。
3. **Double-send 回归测试**（4）：验证 step_completed/run_completed/llm.delta 不触发 Bus 发送，tool 事件正常发送。

### 依赖更新

- `qqbot_adapter/requirements.txt`：websockets 最低版本从 `>=14.1` 调整为 `>=14.0`（实际 install 到 16.0）
