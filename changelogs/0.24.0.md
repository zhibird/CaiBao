# v0.24.0 - 2026-06-02

## 前情与承接

`v0.24.0` 承接 `v0.23.4` 的 QQ Bot 适配器 Phase 4 交付，本版本聚焦三个方向：**(1) Agent 工具扩展**（联网搜索 + 记忆管理 + 聊天记录回溯）、**(2) LLM 消息格式兼容性**（多轮 tool loop 中 content 非字符串类型归一化）、**(3) 安全加固**（QQ Bot 通道守卫 + 密钥防泄漏）。

## 版本概览

1. **Exa MCP 联网搜索** — 集成 Exa MCP 公开端点，无需 API Key，免费使用，默认 provider 切换为 exa
2. **内置 Agent 工具** — web_search / web_fetch 注册为第一公民 ToolDefinition，Agent 开箱即用
3. **记忆管理工具** — forget_memory / recall_memory / message_lookup 三个新工具 + Persona Prompt 人格提示词系统
4. **LLM 消息归一化** — 4 轮迭代修复，覆盖 dict/object/list/non-string 四种非标 content 类型，兼容多轮 tool loop
5. **安全加固** — QQ Bot 守卫收紧 + config.toml 取消跟踪 + 示例配置模板

## 新增功能

### Exa MCP 联网搜索

参考 akashic-agent 的 web_search.py 实现，集成 Exa MCP 公开端点：
- 端点：`https://mcp.exa.ai/mcp` → JSON-RPC 2.0 over SSE
- 无需 API Key，免费使用
- 默认 provider 从 `disabled` 切换为 `exa`

### 内置 Agent 工具

将 web_search / web_fetch 注册为 Agent 的第一公民工具（`AGENT_TOOL_DEFINITIONS`）：
- `web_search`：Brave/Tavily 搜索引擎 (safe)
- `web_fetch`：网页抓取 + HTML 转文本 + 内网 IP 拦截 (safe)
- Handler 复用现有 `web_tools.py` 实现，仅补充 ToolDefinition 包装

### 记忆管理工具 + Persona Prompt

新增 akashic-style Agent 工具：
- `forget_memory`：按 memory_id 删除记忆卡 (dangerous)
- `recall_memory`：关键词搜索记忆卡 (safe)
- `message_lookup`：查看会话聊天记录 (safe)

新增 `app/services/persona_prompt.py`：用户自定义人格提示词系统。

## LLM 消息格式归一化

多轮 tool loop 中，LLM API 要求 `messages[].content` 为 `string` 或 typed content-parts array，但实际出现了 dict/map/untyped-list 等非标格式。本版本经过 4 轮迭代修复：

| 轮次 | 问题 | 根因 | 修复 |
|------|------|------|------|
| 1 | `messages[3]: invalid type: map` | tool call 结果以 dict 形式存在 | 反向判断 `isinstance(content, str)`，覆盖所有非字符串类型 |
| 2 | 多轮 tool loop 中某些消息 content 为 dict | 只处理了 list，未覆盖 dict | 新增 `_normalize_messages` 处理 dict content |
| 3 | 非 string 且非 list 的 content 漏网 | 覆盖范围不完整 | 扩展覆盖：dict/object/自定义类型统一 JSON 序列化 |
| 4 | untyped list-of-dicts 被拒绝 | 缺少 type/text 字段 | 归一化同时补齐必需字段；修复 tool_calls str↔list 转换 |

## 安全加固

### QQ Bot 守卫收紧
- qqbot 通道：收紧权限校验逻辑
- admin 通道：收紧管理接口守卫

### 密钥防泄漏
- `config.toml` 不再被 git 跟踪，防止 `app_id`/`client_secret` 等密钥泄露
- 新增 `config.example.toml` 作为安全模板，开发者复制后填入实际值

## 缺陷修复

### QQ Bot 官方回复恢复
修复 QQ 官方 Bot 通道在某些条件下 reply 不生效的问题。

### 登录字段修正
CaiBao 登录接口使用 `user_id`（非 `account_id`），修正认证 schema 对齐。

### 导入路径修复
`python main.py` 直接运行时 `sys.path` 不含父目录，导致 `qqbot_adapter.xxx` 导入失败。在 `main.py` 头部注入父目录到 `sys.path` 解决。

## 配置变更

### 新增配置项
```toml
# config.toml (从 config.example.toml 复制)
web_search_provider = "exa"        # 默认搜索引擎（原 disabled）

[message]
show_sync_processing_notice = true  # 同步模式是否发送处理提示
processing_notice_delay_seconds = 6.0  # 处理提示延迟（默认 6 秒）
sync_processing_notice = "⏳ 稍等一下，我在认真整理思路中 (ง •̀_•́)ง"
```

### 兼容性说明
- `web_search_provider` 默认值从 `disabled` 改为 `exa`，已有显式配置不受影响
- 消息内容归一化对上游调用方透明，不影响现有 API 契约
- QQ Bot 通道行为不变，安全收紧仅影响边界条件
