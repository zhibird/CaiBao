# v0.24.6 - 2026-06-09

## 前情与承接

`v0.24.6` 承接 `v0.24.5` 的 QQ Bot markdown 格式修复，本版本聚焦三个方向：**(1) B站/Bilibili 专用工具集成**（对标 web_search 的垂直领域增强）、**(2) Agent 执行可靠性**（工具调用预算 + 对话历史注入）、**(3) 模型兼容性**（reasoning_content 传递 + 消息规范化）。

## 版本概览

1. **B站 UP 主最新视频查询** — `bilibili_latest_videos` 工具，5 级回退策略 + 600s 缓存，web_search 自动路由 B站查询
2. **工具调用预算管理** — `max_tool_calls` 防止 Agent 无限循环，同步与流式 tool loop 均支持
3. **推理内容传递** — streaming 支持 `reasoning_delta`，适配 DeepSeek 等 reasoning 模型
4. **对话历史注入** — Agent 支持 `conversation_id` 拉取历史消息注入上下文
5. **消息规范化** — `_assistant_tool_call_message` 统一构建，arguments 强制序列化为 JSON 字符串

## 新增功能

### B站/Bilibili 最新视频查询

新增 `bilibili_latest_videos` 专用工具，解决通用搜索引擎无法准确获取 UP 主最新视频排序的问题：

- **输入**：`up_name`（UP 主名称）或 `mid`（B站 UID），`limit`（1-20，默认 5）
- **mid 解析**：优先 B站搜索 API → Exa 兜底搜索 `space.bilibili.com/{uid}`
- **5 级视频获取回退**：
  1. WBI 签名 API（`/x/space/wbi/arc/search`）
  2. 明文 API（`/x/space/arc/search`）
  3. 动态 feed（`/x/polymer/web-dynamic/v1/feed/space`）
  4. App API（`app.bilibili.com/x/v2/space/archive/cursor`）
  5. RSSHub（3 个镜像：rsshub.app / rsshub.rssforever.com / rsshub.rss.tips）
- **Exa 兜底**：`site:bilibili.com/video` 搜索作为低置信度候选
- **600 秒进程内缓存**，减少重复 API 调用
- **web_search 自动路由**：检测 B站/B站/哔哩 + 最新/投稿/video 等关键词，自动调用 `bilibili_latest_videos`

### 工具调用预算管理

防止 Agent 陷入无限 tool loop：

- 新增 `max_tool_calls` 参数（默认 5，上限 `_MAX_TOOL_LOOP_ITERATIONS`）
- 同步 (`_run_tool_loop`) 和流式 (`_stream_tool_loop`) 均支持
- 达预算后注入 `_tool_budget_prompt` 提示，移除 tools 参数让 LLM 基于已有结果作答
- 每轮迭代检查 `tool_calls_used >= tool_budget`，中间 tool call 处理也检查

## 主要变更

### reasoning_content 传递

适配 DeepSeek 等支持 reasoning 的模型：

- `LLMStreamChunk` 新增 `reasoning_delta: str | None` 字段
- `_parse_stream_chunk` 从 `delta.reasoning_content` 提取 reasoning 增量
- `_build_tool_calls` 和 `_assistant_tool_call_message` 传递 `_assistant_reasoning_content`
- `_ensure_content_is_string` 在消息清理时保留 `reasoning_content`
- 序列化时过滤 `_assistant_` 前缀的内部字段

### 对话历史注入

Agent 支持基于 `conversation_id` 拉取历史消息：

- 新增 `_build_agent_conversation_messages()` 方法
- 从 `chat_history_service.list_messages_for_context()` 获取历史
- 按 channel 过滤（ask/echo/agent/app/qqbot）
- 受 `llm_history_turns` 配置控制最大轮数

### 消息构建规范化

- 新增 `_normalize_tool_call_for_message()` — 清理内部字段，确保 arguments 为 JSON 字符串
- 新增 `_assistant_tool_call_message()` — 统一构建 assistant tool_calls 消息
- 替换所有手动 `{"role": "assistant", "content": None, "tool_calls": [...]}` 构建
- arguments 类型修复：支持 dict/str/None 三种形态归一化
